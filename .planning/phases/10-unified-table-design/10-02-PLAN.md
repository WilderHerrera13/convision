# Plan 10-02: Admin entity lists batch 1 — Brands, Categories, Products, CashTransfers

**Phase:** 10 — Unified Table Design — Entity Tables
**Plan:** 02 of 05
**Wave:** 2
**Depends on:** Plan 10-01 (EntityTable defaults must be updated first)
**Estimated scope:** 4 pages to migrate

---

## Goal

Migrate four admin entity list pages to use the centralized `EntityTable` component, removing all custom pagination state, direct `useQuery` table fetching, and standalone toolbar markup. Each page will render its list through `EntityTable` with `toolbarLeading` (title + count) and `toolbarTrailing` (CTA + filters).

## Success Criteria

- [ ] `Brands.tsx` uses `<EntityTable>` — no local `page`/`search`/`useQuery` for the list
- [ ] `Categories.tsx` uses `<EntityTable>` — no local `page`/`search`/`useQuery` for the list
- [ ] `Products.tsx` uses `<EntityTable>` — no local `page`/`search`/`useQuery` for the list
- [ ] `CashTransfers.tsx` uses `<EntityTable>` — no local `page`/`search`/`useQuery` for the list
- [ ] All four pages compile without TypeScript errors
- [ ] `grep -n "useQuery\|useState.*page\|useState.*search" convision-front/src/pages/admin/Brands.tsx` returns 0 matches for table-related state (page-level dialog state is OK)
- [ ] Each page's sort buttons, search, and Figma-style pagination function correctly (visible in browser at localhost:4300)

      search: params.search,
