**Phase:** 6 — Cash Register Close Module

## Goal

Crear las vistas del asesor para: registrar el cierre de caja diario con medios de pago y conteo físico de efectivo, ver el historial de cierres, registrar el **reporte de gestión diario** y el **registro rápido de atención** según Figma, y ver el historial de reportes.

**Diseño:** Antes de implementar las pantallas del informe y el wizard, seguir **`.planning/phases/06-.../06-FIGMA-AND-FTC.md`**: Figma MCP (`623:2`, frames en tabla) + flujo **`/ftc`**: invocar MCP **`get-ftc-instructions`** (shadcn/studio) cuando esté disponible, luego mapear a `convision-front` (Vite + shadcn/ui; no `next/image`).

## Success Criteria

- [ ] El asesor puede abrir el formulario de cierre, ingresar valores por cada medio de pago y ver la diferencia calculada en tiempo real
- [ ] El conteo de efectivo por denominaciones suma automáticamente
- [ ] Al enviar el cierre, el estado cambia a "Enviado" y el formulario queda bloqueado
- [ ] El historial muestra los cierres anteriores con estado y diferencia total
- [ ] El **reporte de gestión diario** replica secciones y flujos del Figma P3 (`399:286`): Atención (H/M/N), Operaciones, Redes, observaciones, estado/badge, selector coherente con API; sidebar con ítem **«Reporte de gestión diario»** resaltado en la ruta activa
- [ ] **Registro rápido de atención** implementa el wizard de 3 pasos (Figma `634:90`, `698:177`, `698:234`) y actualiza/incrementa métricas vía API
- [ ] Todos los textos en español, sin inline styles, usando EntityTable y DatePicker del proyecto; CTAs con color primario de **recepción** según DS (`33:2`)

  <action>
    **cashRegisterCloseService.ts:**
    ```typescript
    import api from './api'; // axios instance con interceptors

    export interface PaymentMethodEntry {
      name: string;
      registered_amount: number;
      counted_amount: number;
    }

    export interface DenominationEntry {
      denomination: number;
      quantity: number;
    }

    export interface CreateCashClosePayload {
      close_date: string; // YYYY-MM-DD
      payment_methods: PaymentMethodEntry[];
      denominations?: DenominationEntry[];
    }

    export const PAYMENT_METHODS = [
      'efectivo', 'voucher', 'bancolombia', 'daviplata', 'nequi',
      'addi', 'sistecredito', 'anticipo', 'bono', 'pago_sistecredito'
    ] as const;

    export const DENOMINATIONS = [100000, 50000, 20000, 10000, 5000, 2000, 1000, 500, 200, 100, 50];

    const cashRegisterCloseService = {
      list: (params?: Record<string, unknown>) =>
        api.get('/cash-register-closes', { params }),

      get: (id: number) =>
        api.get(`/cash-register-closes/${id}`),

      create: (data: CreateCashClosePayload) =>
        api.post('/cash-register-closes', data),

      update: (id: number, data: Partial<CreateCashClosePayload>) =>
        api.put(`/cash-register-closes/${id}`, data),

      submit: (id: number) =>
        api.post(`/cash-register-closes/${id}/submit`),

      approve: (id: number, admin_notes?: string) =>
        api.post(`/cash-register-closes/${id}/approve`, { admin_notes }),
    };

    export default cashRegisterCloseService;
    ```

    **dailyActivityReportService.ts:** similar pattern con list/get/create/update para /daily-activity-reports
    Incluir los tipos TypeScript para todos los campos del reporte.
