# Plan 12-03 — Status Sync: delivered → Sale completed

## Goal
When a `LaboratoryOrder` is transitioned to `"delivered"` status via `UpdateLaboratoryOrderStatus`, check whether the linked Sale has a zero balance. If it does, mark the Sale as `"completed"` automatically.

## must_haves
- `UpdateLaboratoryOrderStatus` handler in `handler_laboratory.go` calls `h.sale.GetByID` after a successful status update when `input.Status == "delivered"` and `labOrder.SaleID != nil`
- If `sale.Balance <= 0`, the handler calls `h.sale.Update(saleID, salesvc.UpdateInput{Status: "completed"})`
- A zap log is emitted on each sync attempt (success and skip)
- No changes to the `laboratory.Service` or its interface — all orchestration stays in the handler layer

## verification
- `rg "delivered" convision-api-golang/internal/transport/http/v1/handler_laboratory.go` → matches inside `UpdateLaboratoryOrderStatus`
- `rg "h\.sale\.GetByID" convision-api-golang/internal/transport/http/v1/handler_laboratory.go` → matches
- `rg "completed" convision-api-golang/internal/transport/http/v1/handler_laboratory.go` → matches on the `Update` call
- `cd convision-api-golang && go build ./...` → exits 0

Replace with:

```go
o, err := h.laboratory.UpdateOrderStatus(id, input, claims.UserID)
if err != nil {
    respondError(c, err)
    return
}

if input.Status == "delivered" && o.SaleID != nil {
    saleID := *o.SaleID
    sale, saleErr := h.sale.GetByID(saleID)
    if saleErr == nil {
        if sale.Balance <= 0 {
            _, _ = h.sale.Update(saleID, salesvc.UpdateInput{Status: "completed"})
            h.logger.Info("sale marked completed after lab order delivery",
                zap.Uint("sale_id", saleID),
                zap.Uint("lab_order_id", o.ID))
        } else {
            h.logger.Info("lab order delivered but sale has outstanding balance, skipping completion",
                zap.Uint("sale_id", saleID),
                zap.Float64("balance", sale.Balance))
        }
    }
}

c.JSON(http.StatusOK, o)
```

If `h.logger` does not exist on Handler, use a no-op for logging (skip the zap calls, just call the sale update).

**Check handler.go** first: search for `logger` in the Handler struct. If it's not there, omit the `h.logger` calls and replace with a simple inline comment noting the log would go here.
