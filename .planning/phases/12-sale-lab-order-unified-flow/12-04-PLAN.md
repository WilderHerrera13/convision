# Plan 12-04 — Frontend Changes

## Goal
Remove the standalone "Orders (ORD-xxx)" route from `App.tsx`, update the `LaboratoryOrder` TypeScript interface to expose the linked `sale.sale_number` as the primary display reference, and update the lab order detail pages to show `sale_number` instead of `order_number` in the header badge. Update `NewLabOrder` so it accepts `product_type` + `name` instead of requiring only a lens.

## must_haves
- The route `path: "orders"` with `element: <OrderList />` is removed from the receptionist route tree in `App.tsx`
- The `import OrderList from "./pages/receptionist/OrderList"` line is removed from `App.tsx`
- `laboratoryOrderService.ts` — `LaboratoryOrder.sale` type is updated to include `sale_number: string`
- `LaboratoryOrderDetail.tsx` (admin) shows `sale.sale_number` when `sale_id` is set, falling back to `order_number` otherwise
- `LabOrderDetail.tsx` (receptionist) shows `sale.sale_number` when `sale_id` is set, falling back to `order_number` otherwise
- `NewLabOrder.tsx` allows `product_type` and `name` fields when creating a lab order with no specific lens item
- The Orders navigation link (path `/orders` or label "Pedidos"/"Orders") is removed from the admin sidebar in `AdminLayout.tsx`

## verification
- `rg "OrderList" convision-front/src/App.tsx` → no match
- `rg 'path: "orders"' convision-front/src/App.tsx` → no match
- `rg "sale_number" convision-front/src/services/laboratoryOrderService.ts` → matches in the `sale` sub-interface
- `rg "orders\|OrderList" convision-front/src/layouts/AdminLayout.tsx` → returns no match on any nav link
- `cd convision-front && npm run build` → exits 0 (TypeScript build)
