<threat_model>
## Threat Model

**Threats considered:**
- Tests using real bcrypt hashing making tests slow â€” **Mitigation:** Use `bcrypt.GenerateFromPassword` with `bcrypt.MinCost` in test fixtures; `MinCost` is designed for testing and runs ~10x faster than the default cost.
- Test leaking real JWT secrets through assertion messages â€” **Mitigation:** Use `"test-secret-value"` as `JWT_SECRET` set via `os.Setenv` in `TestMain`; this value has no relation to production secrets.
- Mock return values being nil causing panics in service code that dereferences pointers â€” **Mitigation:** Every mock returning a pointer must return either a valid non-nil pointer or explicitly `nil` alongside a non-nil error; never return `(nil, nil)` for pointer-returning methods.
- `time.Now()` calls in `appointment.Service.Create` (scheduled_at in past check) causing flaky tests â€” **Mitigation:** Tests that exercise the "scheduled in past" validation pass `time.Now().Add(-24 * time.Hour)`; tests for success pass `time.Now().Add(72 * time.Hour)`.

**Risk level:** Low (test code only, no production access)
</threat_model>
