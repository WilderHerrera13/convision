# Plan 09-01: Test Infrastructure

## Objective

Create the `internal/testutil/` package with all shared test utilities, hand-written mock implementations for every domain Repository interface, and HTTP test helpers needed by plans 09-02 through 09-05. This plan is wave 1 — all subsequent plans depend on it.

## must_haves
- [ ] `github.com/stretchr/testify v1.11.1` appears as a direct dependency in `go.mod` (no `// indirect`)
- [ ] `go build ./convision-api-golang/internal/testutil/...` exits 0 with no errors
- [ ] `internal/testutil/auth.go` exports `MakeTestClaims`, `InjectClaims`, `AdminClaims`, `SpecialistClaims`, `ReceptionistClaims`, `LaboratoryClaims`
- [ ] `internal/testutil/gin.go` exports `NewRequest`, `DoRequest`, `NewTestRouter`
- [ ] `internal/testutil/assertions.go` exports `AssertStatus`, `AssertJSONField`, `AssertJSONHasKey`
- [ ] `internal/testutil/mocks/` contains at least 15 mock files, each with a compile-time interface assertion `var _ domain.XxxRepository = (*MockXxxRepository)(nil)`

<threat_model>
## Threat Model

**Threats considered:**
- Test files accidentally exposing real credentials in fixtures — **Mitigation:** Use fake IDs and placeholder emails (`test@example.com`) in all test data; never read from `.env` or production env vars in test helpers.
- Mock implementations silently diverging from real interfaces — **Mitigation:** Every mock struct includes a compile-time interface check; the compiler enforces exact method signatures.
- Parallel tests sharing `os.Setenv` for `JWT_SECRET` — **Mitigation:** `JWT_SECRET` is set in each package's own `TestMain`; testutil helpers do not call `os.Setenv` themselves.
- `testutil` accidentally importing `platform/storage/postgres` (violates 3-layer architecture) — **Mitigation:** `testutil` imports only `internal/domain`, `internal/platform/auth`, and stdlib; import cycle caught at `go build` time.

**Risk level:** Low (test-only code, no production data access, no DB connection)
</threat_model>

```go
package testutil

import (
    "encoding/json"
    "net/http/httptest"
    "testing"

    "github.com/stretchr/testify/assert"
)

// AssertStatus asserts recorder.Code == expectedCode.
// On failure, prints the response body to aid debugging.
func AssertStatus(t *testing.T, w *httptest.ResponseRecorder, expectedCode int) {
    t.Helper()
    assert.Equal(t, expectedCode, w.Code, "response body: %s", w.Body.String())
}

// AssertJSONField asserts that the top-level JSON field equals expected.
func AssertJSONField(t *testing.T, w *httptest.ResponseRecorder, field string, expected any) {
    t.Helper()
    var body map[string]any
    if err := json.Unmarshal(w.Body.Bytes(), &body); err != nil {
        t.Fatalf("response is not JSON: %s", w.Body.String())
    }
    assert.Equal(t, expected, body[field])
}

// AssertJSONHasKey asserts that the top-level JSON key exists (any value).
func AssertJSONHasKey(t *testing.T, w *httptest.ResponseRecorder, field string) {
    t.Helper()
    var body map[string]any
    if err := json.Unmarshal(w.Body.Bytes(), &body); err != nil {
        t.Fatalf("response is not JSON: %s", w.Body.String())
    }
    _, ok := body[field]
    assert.True(t, ok, "expected JSON key %q to exist; body: %s", field, w.Body.String())
}
```
