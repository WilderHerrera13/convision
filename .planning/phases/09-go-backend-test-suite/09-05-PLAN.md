<threat_model>
## Threat Model

**Threats considered:**
- `v1.Handler` takes concrete service structs, not interfaces â€” handler tests cannot substitute mock services directly; must wire real services backed by mock repos â€” **Mitigation:** Each handler test builds real service instances (e.g., `patient.NewService(mockPatientRepo, zap.NewNop())`) and passes them to `v1.NewHandler`. This tests both handler parsing and service logic simultaneously.
- `v1.NewHandler` takes 30+ arguments â€” constructing it in every test is verbose â€” **Mitigation:** Create a `buildTestHandler(...)` helper function within the `v1` test package (`package v1_test`) that accepts only the services being tested and passes `nil` for the rest (or zero-value constructed services that will never be called).
- `DashboardRepository` is a concrete type (`*postgresplatform.DashboardRepository`), not an interface â€” cannot mock it â€” **Mitigation:** Pass `nil` for `dashboardRepo` in handler construction for all tests that do not exercise dashboard endpoints. Dashboard-specific handler tests are skipped (`t.Skip("requires real DB â€” DashboardRepository is not an interface")`) in this phase.
- Role guard tests must mount routes with the actual `jwtauth.RequireRole` middleware â€” if test router bypasses all middleware, role guards cannot be tested â€” **Mitigation:** For role guard tests, mount only the specific route under test with `jwtauth.RequireRole` applied, then inject claims with the wrong role via `testutil.InjectClaims`.
- `auth.Service.Login` uses bcrypt and `jwtauth.GenerateToken` (reads `JWT_SECRET` env var) â€” handler tests that hit `POST /api/v1/auth/login` need `JWT_SECRET` set â€” **Mitigation:** `TestMain` in the `v1_test` package sets `JWT_SECRET` and `JWT_TTL_HOURS`.
- Data races from parallel handler tests if shared services are used â€” **Mitigation:** Each test function creates its own service and mock repo instances; no package-level state shared between tests.

**Risk level:** Low-Medium (handler tests exercise the full HTTP layer; no real DB or external services)
</threat_model>
