# Plan 09-03: Unit Tests — Commercial Services

## Objective

Write comprehensive unit tests for the four commercial domain services: product, discount, quote, and sale. These services contain the most complex business logic in the codebase — discount selection, price calculation, quote-to-sale conversion, payment status derivation, and sale cancellation. Tests use mocks from `internal/testutil/mocks/` and verify both the happy path and every business-rule branch.

<threat_model>
## Threat Model

**Threats considered:**
- `sale.derivePaymentStatus` is unexported — cannot test it directly — **Mitigation:** Test indirectly via `Create` (pass payments summing to full total → expect `PaymentStatus == "paid"`) and `AddPayment` (partial sum → `"partial"`).
- `quote.ConvertToSale` touches both `QuoteRepository` and `SaleRepository` — if mock expectations for the second repo are missing, the test will panic — **Mitigation:** Set up both mock repos with explicit `On` expectations before calling `ConvertToSale`; use `mock.AssertExpectations(t, repo)` at test end.
- `product.CalculatePrice` calls `DiscountRepository.GetBestForProduct` — the mock must return both nil and non-nil discount cases — **Mitigation:** Two dedicated test functions with explicit mock return values for each case.
- Floating-point equality assertions — `amountPaid >= total` uses float comparison — **Mitigation:** Use `assert.InDelta` with delta 0.001 for float assertions, not `assert.Equal`.

**Risk level:** Low (test code only, no production access)
</threat_model>
