# Plan 07-01: Figma — Bodegas: Lista, Nueva Bodega, Detalle Bodega

**Phase:** 7 — Figma Bodega & Inventory Module Design
**Plan:** 01 of 04
**Wave:** 1 (foundation — establishes design pattern for all subsequent screens)
**Depends on:** Nothing (first plan)

---

## Goal

Crear en la página Figma "bodega" las tres pantallas base del módulo de Bodegas: lista de bodegas, formulario de creación y pantalla de detalle/edición. Estas pantallas establecen el patrón visual (Admin role, Admin blue `#3A71F7`) y estructural que replicarán los planes 07-02, 07-03 y 07-04.

**Antes de ejecutar:** Leer `pixel.md` completo. Leer `docs/laravel-warehouse-inventory-module.md` secciones 1.1 y 3 (Warehouses endpoints).

---

## Success Criteria

- [ ] Página "bodega" existe en el archivo Figma y contiene exactamente 3 frames al terminar este plan
- [ ] Frame `Lista de Bodegas` — 1440x938px con Sidebar Admin (83:2), tabla contextualizada al módulo, CTA "+ Nueva Bodega" en azul Admin (#3A71F7), 5 filas de datos de bodegas reales, badge Activo/Inactivo correcto
- [ ] Frame `Nueva Bodega` — 1440x938px pantalla completa (NO modal) con FormCard 780px + AsidePanel 332px, campos: Nombre*, Código*, Dirección, Ciudad, Estado, Notas; botones en Topbar; Card/Tip con bg `#EFF1FF` borde `#3A71F7`
- [ ] Frame `Detalle Bodega` — 1440x938px pantalla completa con tabs: "Información" (formulario prefilled) + "Ubicaciones" (tabla de ubicaciones de esa bodega); botones Cancelar + Guardar Cambios en Topbar
- [ ] Todos los iconos de UI son Lucide desde Assets, color `#7D7D87` por defecto
- [ ] No hay emojis en ninguna capa de texto
- [ ] NavItem "Inventario" marcado como activo en el Sidebar Admin
