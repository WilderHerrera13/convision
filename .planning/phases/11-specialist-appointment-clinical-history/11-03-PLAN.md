## Goal

Wire the new service methods into HTTP handlers and register all routes with
correct RBAC. Specialist agenda endpoint, appointment lifecycle completion,
conflict response, and all clinical record CRUD + sign endpoints must be
reachable and return the correct HTTP status codes.

## Success Criteria

- [ ] `GET /api/v1/appointments/active` returns 200 with in_progress appointment or 404
- [ ] `POST /api/v1/appointments/:id/complete` returns 200 on success, 422 on wrong status
- [ ] `POST /api/v1/appointments/:id/take` returns 409 with `{ conflict: true, active_appointment_id }` when conflict
- [ ] `POST /api/v1/appointments/:id/clinical-record` creates a record, returns 201
- [ ] `PUT /api/v1/appointments/:id/clinical-record/anamnesis` upserts, returns 200

- [ ] `PUT /api/v1/appointments/:id/clinical-record/diagnosis` upserts, returns 200
- [ ] `PUT /api/v1/appointments/:id/clinical-record/prescription` upserts, returns 200

- [ ] `GET /api/v1/appointments/:id/clinical-record` returns full record with all sub-entities
- [ ] Follow-up variants: `PUT .../follow-up/anamnesis`, `.../follow-up/evolution`, `.../follow-up/formula`
- [ ] All clinical record endpoints: `RequireRole(RoleSpecialist, RoleAdmin)`
- [ ] `make build` passes with 0 errors

    ```go
        id, err := strconv.ParseUint(c.Param("id"), 10, 64)
        if err != nil {

                })

            }
            respondError(c, err)
            return
        }
        c.JSON(http.StatusOK, toAppointmentResource(a))
    }
    ```

    ```go
    func (h *Handler) UpsertFollowUpEvolution(c *gin.Context)

    // PUT /appointments/:id/clinical-record/follow-up/formula
    func (h *Handler) UpsertFollowUpFormula(c *gin.Context)
    ```

    Each handler follows the same thin pattern:
    1. Parse `:id` → `appointmentID uint`
    2. Bind JSON body to the service DTO
    3. Call the service method (passing `clinicID` from JWT claims if needed)
    4. Map error with `respondError(c, err)` or return JSON

    **`CreateClinicalRecord`** reads `record_type` from JSON body and calls
    `h.clinicalRecord.CreateRecord(clinicID, input)`. Returns 201.

    **`SignAndCompleteClinicalRecord`** reads `professional_tp` from JSON body,
    gets `specialistID` from JWT claims, calls the correct service (`clinicalRecord`
    or `followUp`) based on the record's `record_type`. Returns 200.
    First fetches the record to determine type, then routes to the correct `SignAndComplete`.

    Use `c.ShouldBindJSON` for all body parsing. Return 422 for validation errors.
