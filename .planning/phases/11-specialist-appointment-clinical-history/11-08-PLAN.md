# Plan 11-08: Frontend — Prescription Legal Document Preview + Digital Signature

**Phase:** 11 — Specialist Appointment & Clinical History Flow
**Plan:** 08 of 08
**Wave:** 8
**Depends on:** 11-06, 11-07 (both consultation flows must exist — this plan wires their "Firmar" buttons)
**Estimated scope:** 3 new files + 2 modified files

---

## Goal

Build the legal optical prescription document preview (Figma node 2314:2136)
and the digital signature confirmation modal. When the specialist clicks
"Firmar y completar consulta" or "Firmar y cerrar control", they see the
legal document preview, review it, enter their T.P., and confirm — which
calls the backend `/sign` endpoint, marks the appointment `completed`, and
redirects to the agenda.

---

## Success Criteria

- [ ] `PrescriptionPreviewModal` renders a styled legal document matching Figma 2314:2136
- [ ] Preview shows: clinic header, patient data, professional data, OD/OI formula table, lens type + treatments, validity date, CUPS code, signature block
- [ ] "Confirmar y firmar" button in the modal calls `signClinicalRecord(appointmentId, professionalTp)`
- [ ] On sign success: appointment status → `completed`; navigate to `/specialist/appointments`
- [ ] `DigitalSignatureModal` (inner confirmation step) shows T.P. input + legal consent text
- [ ] After successful sign: toast "Consulta completada y firmada correctamente"
- [ ] Print/export button triggers `window.print()` on the preview content (styled for print via CSS)
- [ ] `npm run build` passes with 0 TypeScript errors

## Verification

Full end-to-end flow test (dev server must be running with backend):

1. Log in as `specialist@convision.com`
2. Navigate to `/specialist/appointments` — agenda loads
3. Take a scheduled appointment → timer starts
4. Click "Iniciar Historia Clínica" → 4-tab form opens
5. Fill all 4 tabs with test data
6. Tab 4 → "Firmar y completar consulta" → prescription preview modal opens
7. Preview shows patient name, OD/OI values, CUPS 890205, validity date
8. Enter T.P. CTNPO → check legal confirmation checkbox → "Firmar y completar"
9. Toast appears: "Consulta completada y firmada correctamente"
10. Redirected to `/specialist/appointments` — appointment shows status "Completada"

## Must Haves

- `formatOptical(0)` returns `'Plano'` — not `'+0.00'`
- Sign confirmation is a 2-step confirmation: "Firmar" (preview) → "Confirmar y firmar" (T.P. + checkbox)
- After sign: appointment status in the list must refresh to `completed` (invalidate React Query cache)
- Print button: generates print-ready prescription (CSS `@media print` hides everything except the document)
- Legal texts visible on document: "Ley 650/2001 Art. 24", "Decreto 2200/2005", CUPS code, validity date
