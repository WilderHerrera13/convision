    Unimplemented methods should `panic("not implemented")` or return zero values.

    Mock structure:
    ```go
    type mockAppointmentRepo struct {
        appointments map[uint]*domain.Appointment
        nextID       uint
    }

    func newMockRepo() *mockAppointmentRepo {
        return &mockAppointmentRepo{appointments: make(map[uint]*domain.Appointment), nextID: 1}
    }

    func (m *mockAppointmentRepo) GetByID(id uint) (*domain.Appointment, error) { ... }
    func (m *mockAppointmentRepo) Update(a *domain.Appointment) error { ... }
    func (m *mockAppointmentRepo) GetActiveBySpecialist(specialistID uint) (*domain.Appointment, error) {
        for _, a := range m.appointments {
            if a.TakenByID != nil && *a.TakenByID == specialistID && a.Status == domain.AppointmentStatusInProgress {
                return a, nil
            }
        }
        return nil, &domain.ErrNotFound{Resource: "appointment"}
    }
    // All other interface methods: return nil, nil or panic
    ```

    **Test cases** (table-driven using `t.Run`):

    1. `TestTake_Success` — scheduled appointment → in_progress; `StartedAt` is set
    2. `TestTake_ConflictReturns409Error` — specialist already has in_progress → returns `*ErrActiveAppointment`
    3. `TestTake_SameAppointmentNoConflict` — taking the same already-in-progress appointment (resume scenario) → no conflict error
    4. `TestPause_Success` — in_progress → paused
    5. `TestPause_WrongStatus` — paused appointment → returns ErrValidation
    6. `TestResume_Success` — paused → in_progress
    7. `TestComplete_Success` — in_progress → completed; `CompletedAt` is set
    8. `TestComplete_WrongStatus` — scheduled appointment → returns ErrValidation
    9. `TestGetActive_Found` — returns in_progress appointment for specialist
    10. `TestGetActive_NotFound` — returns ErrNotFound when no active appointment

    Use `require` or `assert` from `testing` stdlib (no testify unless already used in project).
    Check existing test files for the assertion style used.
  </action>
