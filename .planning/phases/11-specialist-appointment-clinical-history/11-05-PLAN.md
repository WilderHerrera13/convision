# Plan 11-05: Frontend — Specialist Agenda + Appointment Detail

**Phase:** 11 — Specialist Appointment & Clinical History Flow
**Plan:** 05 of 08
**Wave:** 5
**Depends on:** 11-03 (API endpoints must be registered; frontend consumes them)
**Estimated scope:** 6 new files + 2 modified files

---

## Goal

Build the specialist agenda page (Figma nodes 2112:517, 2111:459, 2115:655)
and the appointment detail page (nodes 2138:1031, 2251:1416, 2181:1057,
2150:1044). Implement the appointment lifecycle buttons (Tomar, Pausar,
Reanudar, Completar), the live elapsed timer, and the conflict modal.
All pixel-exact against Figma using `#0f8f64` green tokens.

---

## Success Criteria

- [ ] Route `/specialist/appointments` renders `SpecialistAppointmentsPage` — protected to `RoleSpecialist`
- [ ] Route `/specialist/appointments/:id` renders `SpecialistAppointmentDetailPage`
- [ ] Empty state shows KPI cards (Citas hoy / En curso / Completadas) matching Figma 2112:517
- [ ] List shows date filter, patient search, status chips (Programada/En curso/Pausada/Completada/Cancelada) matching Figma 2111:459
- [ ] Each appointment row is an `AppointmentCard` component
- [ ] Detail page shows "Tomar cita" button for scheduled appointments (Figma 2138:1031)
- [ ] "Tomar cita" → 200: status changes to `in_progress`, timer starts
- [ ] "Tomar cita" → 409: `ConflictModal` appears with "Pausar y tomar esta" / "Ir a mi cita activa" (Figma 2150:1044)
- [ ] Detail page for in_progress shows tab bar: Resumen / Historia clínica / Anotaciones / Evolución / Prescripción / Notas internas (Figma 2181:1057)
- [ ] Active timer banner shows patient name + elapsed `HH:MM:SS`
- [ ] Pausar button visible when in_progress; Reanudar when paused
- [ ] `npm run build` passes with 0 TypeScript errors
