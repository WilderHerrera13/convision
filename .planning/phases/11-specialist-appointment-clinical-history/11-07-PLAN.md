# Plan 11-07: Frontend — Follow-up / Control Flow (4 Tabs)

**Phase:** 11 — Specialist Appointment & Clinical History Flow
**Plan:** 07 of 08
**Wave:** 7
**Depends on:** 11-06 (shared layout components + ClinicalFormLayout + ClinicalAsidePanel must exist)
**Estimated scope:** 6 new files + 1 modified file

---

## Goal

Build the 4-tab follow-up/control form at `/specialist/appointments/:id/follow-up`
(Figma nodes 2273:2001, 2253:1472, 2273:2149, 2267:1752, 2267:1920).
The comparative visual exam (Tab 2) shows the previous visit's values in green
alongside the new entry. The diagnostic evolution tab (Tab 3) renders active
diagnosis cards with an evolution radio. The formula update tab (Tab 4) pre-fills
from the last prescription and lets the specialist choose "Mantener" or "Actualizar".

---

## Success Criteria

- [ ] Route `/specialist/appointments/:id/follow-up` renders `ClinicalHistoryFollowUpPage`
- [ ] Tab 1 (Anamnesis Control): motivo control + satisfacción radio + cambios subjetivos + medicamentos + cambios sistémicos + uso corrección + horas diarias + obs — matches Figma 2273:2001

- [ ] Tab 3 (Evolución Diagnóstica): active DX cards (green border) + evolution radio + description + new DX toggle + continuity plan — matches Figma 2267:1752

- [ ] AsidePanel shows: fórmula vigente with expiry warning + DX activo + step counter + marco legal (CUPS 890307)
- [ ] "Firmar y cerrar control" button on Tab 4 triggers sign flow (Plan 11-08)
