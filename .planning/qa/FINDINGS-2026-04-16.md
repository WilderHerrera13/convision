## Hallazgos (FAIL / GAP) — Usuarios

### QA-001
- Rol: admin
- URL: http://localhost:4300/admin/users/create
- Severidad: menor
- Pasos:
  1. Navegar directamente a `/admin/users/create` (URL alternativa que el usuario podría intentar)
  2. Observar resultado
- Esperado: Redirigir a 404 o mostrar página de error amigable
- Observado: React Router interpreta "create" como `:id`, carga `UserDetailPage`, detecta NaN, hace navigate durante el render. El formulario no se muestra. La URL correcta es `/admin/users/new` (definida en App.tsx)
- Evidencia: consola muestra "You should call navigate() in a React.useEffect(), not when your component is first rendered." `UserDetailPage.tsx` línea 52
- Estado: confirmado

### Resolución (2026-04-16)

- Ruta hija `users/create` → `<Navigate to="/admin/users/new" replace />` en `App.tsx` (evita matchear `:id`).
- Estado: **resuelto**

---

### QA-002
- Rol: admin
- URL: http://localhost:4300/admin/users (y todas las rutas del módulo)
- Severidad: mayor
- Pasos:
  1. Abrir la aplicación en un viewport de ancho < 900px (o dispositivo móvil)
  2. Navegar a cualquier sección del panel admin
  3. Intentar interactuar con la tabla o formularios
- Esperado: Layout responsive que adapte el sidebar al viewport (colapsar o convertir en hamburger menu funcional)
- Observado: El sidebar fijo de 240px ocupa el 80% del viewport de 300px, bloqueando completamente el acceso a la tabla de usuarios, botones de acción (Ver, Editar, Eliminar), campos del formulario de creación/edición, y el botón de guardar/crear ubicado en el topbar
- Evidencia: Click interceptado por sidebar en `top=56, left=240, 60x545`. Botones de tabla inaccesibles. El botón "Crear Usuario" en el topbar queda fuera del viewport visible
- Estado: confirmado

### Resolución (2026-04-16)

- `AdminLayout`: en `isMobile` el sidebar usa `openMobile` (antes `toggleSidebar` solo mutaba `openMobile` y el layout leía `state` de `open`). Cierre al navegar, backdrop y `aside` fijo en móvil. Por debajo de 1024px se fuerza `setOpen(false)` para liberar el área principal en tablet.
- Archivos: `convision-front/src/layouts/AdminLayout.tsx`
- Estado: **resuelto**

---

### QA-003
- Rol: admin (y todos los roles)
- URL: http://localhost:4300/login
- Severidad: menor (sugerencia de refactor)
- Pasos:
  1. Ingresar credenciales incorrectas en el login
  2. Hacer click en "Ingresar"
- Esperado: Mostrar mensaje de error inline en el formulario con `loginError` state
- Observado: `AuthContext.login()` maneja el error internamente con un toast (correcto, el usuario ve el feedback) pero **no relanza la excepción**. Por eso el `catch` en `Login.tsx` nunca ejecuta y `loginError` state nunca se llena. El estado `loginError` es código muerto. El usuario SÍ recibe feedback (toast), pero el código es inconsistente
- Evidencia: `AuthContext.tsx` línea 111-116 captura el error con toast sin `throw`. `Login.tsx` línea 62-68 tiene catch que nunca ejecuta. `loginError` (línea 25) nunca se setea desde el flujo normal
- Estado: confirmado

### Resolución (2026-04-16)

- `AuthContext.login`: en error se hace `throw new Error(message)` para que `Login.tsx` rellene `loginError`. Mensaje en línea con `role="alert"`.
- Archivos: `convision-front/src/contexts/AuthContext.tsx`, `convision-front/src/pages/Login.tsx`
- Estado: **resuelto**

---

### QA-004
- Rol: admin
- URL: http://localhost:4300/admin/users/:id (cuando :id no es numérico)
- Severidad: mayor
- Pasos:
  1. Navegar a `/admin/users/create` o cualquier URL con `:id` no numérico
  2. Observar el comportamiento del componente
- Esperado: El redirect a `/admin/users` debe hacerse dentro de `useEffect` para seguir el ciclo de vida de React
- Observado: `UserDetailPage.tsx` llama `navigate()` directamente en el cuerpo del componente (no en useEffect), generando el warning de React Router. Aunque el redirect funciona, viola las reglas de React y puede causar comportamiento inesperado en React StrictMode o futuras versiones
- Evidencia: `UserDetailPage.tsx` líneas 51-54:
  ```tsx
  if (!Number.isFinite(userId)) {
    navigate('/admin/users', { replace: true });  // <-- navigate en el render
    return null;
  }
  ```
  Consola: "You should call navigate() in a React.useEffect(), not when your component is first rendered."
- Estado: confirmado

### Resolución (2026-04-16)

- Redirect en `useEffect` en `UserDetailPage.tsx` y `UserEditPage.tsx` (mismo patrón).
- Estado: **resuelto**
